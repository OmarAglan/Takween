# تكوين (Takween)

> نظام البناء العربي الرسمي للغة **باء (Baa)**.
> 
> The Arabic-first build system for Baa.

## الحالة الحالية (Current Status)
- **آخر تحديث:** 2026-03-02
- **اعتماد باء:** المتطلبات الأساسية لتكوين متاحة في Baa `v0.4.4.0` (ذاكرة ديناميكية + ملفات + معاملات CLI + `نفذ_أمر`).
- **وضع تكوين:** بدء تنفيذ MVP (Week 1) مع تركيز على:
  - `تهيئة / init`
  - محلل صارم لملف `مشروع.تكوين`
  - عقود CLI واضحة قبل استكمال `بناء/تشغيل/تنظيف`

## لماذا تكوين؟
- واجهة أوامر عربية مباشرة لمشاريع Baa.
- ملف إعداد بسيط بصيغة `مفتاح: قيمة`.
- نهج صارم في التحقق المبكر من الأخطاء.
- تصميم موجه للـ dogfooding (بناء أدوات Baa نفسها لاحقاً).

## البدء السريع (Quick Start)
> هذه الواجهة هي العقد الرسمي الجاري تنفيذه حالياً.

> ملاحظة: هذا المستودع يضم نسخة محلية من `stdlib/baalib.baahd` للتوافق مع بعض إصدارات `baa` التي لا تبحث تلقائياً في مسار التثبيت.

```powershell
# تهيئة مشروع جديد (Arabic + English alias)
takween تهيئة
takween init

# بناء المشروع
takween بناء
takween build

# بناء ثم تشغيل
takween تشغيل
takween run

# تنظيف المخرجات
takween تنظيف
takween clean
```

## مثال `مشروع.تكوين` (MVP)
```text
الاسم: تطبيقي_الأول
الإصدار: 1.0.0
النوع: تنفيذي
المصدر: المصدر/
المخرج: بناء/
ملفات: الرئيسية.baa
التحسين: ١
```

## ما هو مُفعّل الآن؟
- `تهيئة/init`: قيد التنفيذ في نواة MVP.
- تحليل `مشروع.تكوين`: صارم (Fail-Fast) مع رسائل خطأ لكل سطر.
- `بناء/تشغيل/تنظيف`: عقد CLI مثبت، مع تنفيذ تدريجي حسب خارطة الطريق.
