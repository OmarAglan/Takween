# عقد CLI (Takween CLI Contract)

## الأوامر الرسمية (MVP)
- `تكوين تهيئة`
- `تكوين init`
- `تكوين بناء`
- `تكوين build`
- `تكوين تشغيل`
- `تكوين run`
- `تكوين تنظيف`
- `تكوين clean`

> alias متوافق: `takween ...`

## السلوك المتوقع

### `تهيئة / init`
- ينشئ قالب مشروع مبدئي:
  - `مشروع.تكوين`
  - `المصدر/`
  - `المصدر/الرئيسية.baa`

### `بناء / build`
- يقرأ `مشروع.تكوين`.
- يفشل مباشرة عند أي خرق للمواصفة.
- ينفّذ بناء فعلي عبر `baa` ويولّد الناتج حسب `المخرج + الاسم + النوع`.

### `تشغيل / run`
- يستدعي build أولاً.
- عند نجاح build: يشغّل الناتج.
- يُرجع كود خروج البرنامج الناتج كما هو.

### `تنظيف / clean`
- يحذف مخرجات البناء فقط (مثل `بناء/`).
- يرفض المسارات غير الآمنة (`.`, الجذر, أو أي مسار يحتوي `..`).

## أكواد الخروج (Exit Codes)
- `0`: نجاح.
- `1`: خطأ تنفيذي عام.
- `2`: خطأ استخدام CLI (أمر/وسائط غير صحيحة).
- `3`: أمر معروف لكنه غير مفعل بالكامل بعد.

## رسائل المساعدة
عند عدم تمرير أمر أو تمرير أمر غير معروف، يجب عرض سطر استخدام واضح يتضمن الأسماء العربية والإنجليزية.
