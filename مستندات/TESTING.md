# الاختبارات (Takween Testing)

## هدف المرحلة
ضمان ثبات سلوك MVP قبل التوسع في البناء الذكي ومدير الحزم.

## طبقات الاختبار
1. **Parser Validation**
2. **CLI Contract**
3. **Init Side Effects**
4. **Manual End-to-End Smoke**

## حالات أساسية

### Parser
- ملف صحيح بالكامل -> نجاح.
- مفتاح غير مدعوم -> فشل.
- حقل إلزامي مفقود -> فشل.
- `التحسين` خارج المجال -> فشل.
- `ملفات` فارغة أو بلا `.baa` -> فشل.

### CLI
- أمر عربي صحيح -> الدالة الصحيحة تُستدعى.
- alias إنجليزي صحيح -> نفس السلوك.
- أمر غير معروف -> كود خروج `2`.

### Init
- إنشاء `المصدر/` إذا لم يوجد.
- إنشاء `مشروع.تكوين` قالب.
- إنشاء `المصدر/الرئيسية.baa` قالب.

## سيناريو Smoke (يدوي)
1. `takween تهيئة`
2. تحقق من الملفات المتولدة.
3. عدّل `مشروع.تكوين` بإدخال خطأ متعمد.
4. نفّذ `takween build` وتأكد من فشل واضح.
5. أصلح الخطأ وأعد `takween build`.

## ملاحظة
حتى إضافة runner رسمي داخل Takween، تعتمد الاختبارات على سيناريوهات يدوية موثقة.
