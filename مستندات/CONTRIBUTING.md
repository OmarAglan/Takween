# المساهمة في تكوين (Takween Contributing)

## نطاق هذا المستودع
هذا المستودع مخصص لأداة Takween نفسها (وليس لمصدر Baa).

- مرجع Baa الرسمي: <https://github.com/OmarAglan/Baa>
- التوثيق المرجعي لباء يبقى في المصدر الرسمي.

## المتطلبات
- Git
- نسخة صالحة من مصرّف Baa على الجهاز
- بيئة Windows (نطاق MVP الحالي)

## ملاحظة stdlib
يتضمن هذا المستودع ملف `stdlib/baalib.baahd` لتوافق التطوير المحلي. إذا تم تحديثه، يجب أن يتطابق مع النسخة الرسمية من Baa.

## طريقة العمل
1. افتح فرعاً جديداً من `main`.
2. نفّذ التعديلات في نطاق واضح (docs أو parser أو CLI).
3. حدّث التوثيق المتأثر (`README`, `ROADMAP`, `مستندات/*`).
4. اختبر سيناريوهات MVP الأساسية (راجع `TESTING.md`).
5. افتح Pull Request مع ملخص واضح.

## معايير قبول PR
- لا تعارض بين السلوك الفعلي والتوثيق.
- الالتزام بعقود CLI في `CLI.md`.
- أي تغيير في الصيغة يجب أن ينعكس في `FORMAT.md`.
- أي تغيير في خطة التنفيذ يجب أن ينعكس في `ROADMAP.md`.

## أسلوب الكود (Baa)
- أسماء دوال واضحة بالعربية.
- رسائل خطأ عربية مباشرة.
- إبقاء دوال parser نقية قدر الإمكان (بدون side effects غير ضرورية).
- عزل استدعاءات النظام في `منفذ_الأوامر.baa`.

## الرسائل والإصدارات
- استخدم رسائل commit مباشرة (مثال: `parser: enforce strict required keys`).
- لا تجمع تغييرات كبيرة غير مترابطة في PR واحد.
