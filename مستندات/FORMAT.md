# صيغة ملف التكوين (Takween Format Specification)

ملف `مشروع.تكوين` مصمم ليكون أبسط بكثير من `CMakeLists.txt` أو `Makefile`. يعتمد على صيغة `مفتاح: قيمة` (Key-Value pair) بدون أقواس معقدة أو دوال.

## القواعد الأساسية (Rules)
1. **التعليقات:** تبدأ بـ `//` ويتم تجاهلها.
2. **المفتاح والقيمة:** يُفصل بينهما بنقطتين رأسيتين `:` متبوعة بمسافة.
3. **القوائم:** تُفصل عناصرها بفاصلة أجنبية `,` أو عربية `،`.

## الحقول المدعومة (Supported Fields)

### الحقول الإجبارية (Required)
* `الاسم`: اسم المشروع أو التطبيق الناتج (بدون مسافات).
* `الإصدار`: رقم إصدار المشروع (مثال: `١.٠.٠` أو `1.0.0`).
* `النوع`: `تنفيذي` (لتوليد .exe) أو `مكتبة` (لتوليد .a/.dll).

### الحقول الاختيارية (Optional)
* `المصدر`: المجلد الذي يحتوي على الكود (الافتراضي: `المصدر/`).
* `المخرج`: المجلد الذي سيتم وضع الناتج فيه (الافتراضي: `بناء/`).
* `التحسين`: مستوى التحسين للمترجم (٠، ١، أو ٢). الافتراضي: `١`.
* `مكتبات`: قائمة بالمكتبات الخارجية المراد ربطها.
* `أعلام_إضافية`: أي أوامر إضافية تريد تمريرها لمترجم باء مباشرة.

## مثال شامل (Full Example)
```text
// إعدادات المشروع الأساسية
الاسم: محرك_ألعاب
الإصدار: ٠.٥.٠
النوع: تنفيذي

// إعدادات المجلدات
المصدر: المصدر/
المخرج: المخرجات_النهائية/

// خيارات التجميع
التحسين: ٢
أعلام_إضافية: --asm-comments

// مكتبات النظام والمكتبات الخارجية
مكتبات: user32، gdi32، مكتبة_الصوت
