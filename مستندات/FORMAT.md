# صيغة ملف `مشروع.تكوين` (Takween Format)

## الهدف
هذه المواصفة تحدد صيغة MVP الحالية، وتُستخدم مباشرة بواسطة parser الصارم في تكوين.

## قواعد عامة
1. الترميز: `UTF-8`.
2. كل سطر فعال يجب أن يكون بصيغة `مفتاح: قيمة`.
3. التعليقات تبدأ بـ `//` من أول السطر.
4. الأسطر الفارغة مسموحة.
5. أي مفتاح غير معروف = **خطأ فوري**.

## المفاتيح المدعومة

### الحقول الإلزامية
- `الاسم`: اسم المشروع الناتج.
- `الإصدار`: إصدار المشروع (نصي، مثل `1.0.0`).
- `النوع`: `تنفيذي` أو `مكتبة`.
- `المصدر`: مجلد المصدر (مثل `المصدر/`).
- `المخرج`: مجلد المخرجات (مثل `بناء/`).
- `ملفات`: قائمة ملفات `.baa` مفصولة بفاصلة.

### الحقول الاختيارية
- `التحسين`: `0|1|2` أو `٠|١|٢`.
- `مكتبات`: قائمة مكتبات للربط لاحقاً.
- `أعلام_إضافية`: أعلام إضافية تمرَّر للمصرّف.

## قواعد `ملفات`
- القيمة إلزامية في MVP.
- يجب أن تحتوي على ملف واحد أو أكثر.
- تُفسَّر المسارات نسبياً إلى `المصدر`.
- مثال:
  - `ملفات: الرئيسية.baa`
  - `ملفات: الرئيسية.baa, أدوات/سلاسل.baa`

## مثال كامل
```text
// Takween MVP project file
الاسم: تطبيق_تجريبي
الإصدار: 1.0.0
النوع: تنفيذي
المصدر: المصدر/
المخرج: بناء/
ملفات: الرئيسية.baa, أدوات/مساعدة.baa
التحسين: ١
```

## أخطاء شائعة (Strict Mode)
- مفتاح غير مدعوم.
- قيمة فارغة لحقل إلزامي.
- `التحسين` خارج المجال المسموح.
- `ملفات` بدون امتداد `.baa`.

## ملاحظات MVP
- لا يوجد directory auto-scan حالياً.
- لا يوجد دعم package fetching حالياً.
- المواصفة ستتطور بالتزامن مع `ROADMAP.md`.
